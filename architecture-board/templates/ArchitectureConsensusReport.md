# Architecture Consensus Report

## Purpose

Consolidate all perspective reviews into a single consensus document with agreed findings, decisions and open items.

## Scope

Synthesis of Domain, Platform, Knowledge, Governance and LLM reviews for one ARB session.

## Required Inputs

| Input | Source |
|-------|--------|
| Architecture Brief | Working folder |
| Domain Architecture Review | Working folder |
| Platform Review | Working folder |
| Knowledge Review | Working folder |
| Governance Review | Working folder |
| LLM Review | Working folder |

## Output Format

Completed report copied to working folder as `ArchitectureConsensusReport.md`.

## Do Not

- Do not treat consensus decisions as official until recorded in ADR/DAR.
- Do not omit dissenting views from the record.
- Do not promote to `docs/` without completed decision package.
- Do not reference unapproved decisions as final.
