# Governance Review

## Purpose

Evaluate security, privacy, compliance, Approval Gate readiness, EduQA alignment and institutional governance impact.

## Scope

Security controls, access policies, audit requirements, quality assurance gates and documentation governance.

## Required Inputs

| Input | Source |
|-------|--------|
| Completed Architecture Brief | Working folder |
| Prior perspective reviews | Working folder |
| Security and QA specifications | `docs/` |
| Risk register entries | Session artifacts |

## Output Format

Completed review copied to working folder as `GovernanceReview.md` with findings, risks and recommendations.

## Do Not

- Do not approve proposals that bypass Approval Gate requirements.
- Do not treat governance review as a substitute for security testing.
- Do not promote content to `docs/` from this review alone.
- Do not reference unapproved decisions as final.
