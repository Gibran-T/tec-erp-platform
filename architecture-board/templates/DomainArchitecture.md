# Domain Architecture Review

## Purpose

Evaluate business domain modeling, educational logic, mission alignment and domain service boundaries for the topic under review.

## Scope

Domain layer concerns: Business Missions, competencies, pedagogical flows, simulation boundaries and business-first design.

## Required Inputs

| Input | Source |
|-------|--------|
| Completed Architecture Brief | Working folder |
| Relevant functional specs | `docs/` |
| Domain diagrams or models | Session artifacts |
| Open domain questions | Brief or prior reviews |

## Output Format

Completed review copied to working folder as `DomainArchitecture.md` with findings, risks and recommendations.
