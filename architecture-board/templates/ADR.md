# Architecture Decision Record (ADR)

## Purpose

Record a single architectural decision with institutional traceability, context, rationale and consequences.

## Scope

One decision per ADR. Immutable once accepted — supersede with a new ADR rather than editing.

## Required Inputs

| Input | Source |
|-------|--------|
| Approved Architecture Decision Package | Working folder |
| Architecture Consensus Report | Working folder |
| Related ADR numbers | Prior records |

## Output Format

Completed ADR copied to working folder as `ADR-NNN-<short-title>.md`.

## Do Not

- Do not edit an accepted ADR — supersede with a new record.
- Do not mark as Accepted without ARB consensus.
- Do not treat Proposed ADRs as binding decisions.
- Do not reference unapproved decisions as final.
