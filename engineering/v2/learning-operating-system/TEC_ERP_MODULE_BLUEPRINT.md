# TEC.ERP Module Blueprint (Reusable Template)

> **Status:** CONTRACT — NOT IMPLEMENTED · Wave 1 Learning Architecture · Do not treat as shipped UX.

**Authority:** TEC.ERP Learning Operating System Master Contract · Owner review required before Wave 2.

---

Every module content pack SHALL populate sections **A–W**. Identical filler across modules is non-compliant.

## Required sections

| ID | Section | Intent |
|----|---------|--------|
| A | Module identity | Code, title FR/EN, curriculum version behavior |
| B | Hero visual concept | One dominant visual idea (process/enterprise), not stock decoration |
| C | Business challenge | Why Equinoxe / learner must care |
| D | Process map | Explicit scope + boundaries |
| E | Departments and actors | Named roles |
| F | Data and documents | Master data + transactional docs |
| G | Concepts and glossary | Acronyms explained |
| H | Quantitative example | Numbers with business meaning |
| I | Common errors | Realistic failure modes |
| J | Demonstration | Professor/ERP demo script |
| K | Mission 1 | Comprendre et cadrer |
| L | Mission 2 | Exécuter et contrôler |
| M | Mission 3 | Analyser, récupérer et décider |
| N | BI Studio | KPI set + learner analysis path |
| O | AI Decision Workspace | Mode(s) + evidence requirements |
| P | Assessment | Item types + unlock |
| Q | Competency evidence | Minimum diverse evidence |
| R | Debrief | Synthesis + next module |
| S | Teaching Deck | 17-slide anchors |
| T | Professor notes | Decisions to elicit in class |
| U | Learner review | Post-class safe review |
| V | Accessibility and language | FR/EN, contrast, no raw enums |
| W | V1/V2 behavior | Historical vs current rules |
| X | Consulting mandat | Sponsor, problem, incomplete evidence, decision (Wave 1A) |
| Y | Process chain contribution | Which E2E chains this module owns/feeds |
| Z | Experience modes | Where Exploration / Guided / Simulation / Evaluation apply |
| AA | Dual journey | Company evolution + professional identity stage |
| AB | Twin / ELE hooks | Twin state touched; stakeholder events if any (pilot) |

Module-specific instances: `modules/M*_LEARNING_BLUEPRINT.md`.
