## Vocabulary

| Term | Meaning |
|------|---------|
| Module | Academic structure (M1–M10) |
| Mandat | Professional assignment from a sponsor |
| Mission | Specific execution unit inside the mandat (exactly 3 regular missions per module) |

## Required mandat fields (every module)

| Field | Requirement |
|-------|-------------|
| Mandate title | Professional title |
| Executive sponsor | Named sponsor role |
| Client problem | Authentic problem |
| Business context | Twin situation |
| Stakeholders | Who is involved |
| Available evidence | What exists |
| Incomplete evidence | Explicit gaps |
| Constraints | Limits |
| Deadline | Urgency |
| Expected deliverables | Artifacts |
| Decision | Decision point |
| Success criteria | What “good” means pedagogically |
| Impact criteria | Which Twin/KPI/stakeholder effects matter |
| Professor facilitation notes | BEFORE/DURING/AFTER hooks |

Opening package MUST also satisfy `TEC_ERP_DECISION_CASE_METHOD_CONTRACT.md` (no theory wall; no revealed answer).

## Three-mission production inside every mandat

### MISSION 1 — Enquête

enquête · diagnostic · hypothesis · evidence map

### MISSION 2 — Intervention

ERP intervention · control · transactional proof

### MISSION 3 — Décision et transfert

consequence analysis · BI interpretation · recovery · executive recommendation

This aligns with `TEC_ERP_THREE_MISSION_PEDAGOGICAL_CONTRACT.md` without replacing it.

## Outputs

Every mandat SHOULD produce:

- Dossier evidence chain  
- Decision defense  
- Executive impact awareness  
- Reflection + reapplication  
- Teaching Deck anchors for Professor  

## Anti-patterns

- Module as menu of unrelated exercises  
- Missions that do not share the same mandat problem  
- Success defined only as points  
- Missing incomplete-evidence declaration
