# TEC.ERP Executive Impact After-Action Contract

> **Status:** CONTRACT — NOT IMPLEMENTED · Wave 1A Research Addendum · Do not treat as shipped UX.

**Authority:** TEC.ERP Learning Operating System Master Contract · Owner review required before Wave 2.

---

## Purpose

After every important mission or decision, show consequence literacy across four time/enterprise levels — without invented precision.

## Four levels

### MAINTENANT

What changed immediately?

### DANS LE PROCESSUS

Which document, department, and control changed?

### DANS L’ENTREPRISE

Which KPI, cost, service, workforce, or risk changed?

### DANS LE TEMPS

What delayed effect may appear?

## Required dimensions (show those affected; do not force fake values)

operational cost · OTIF · inventory · service · margin · productivity · customer satisfaction · cash flow · payables · receivables · workforce · compliance · risk

## Every displayed impact MUST include

| Field | Rule |
|-------|------|
| Baseline | Prior value or state |
| New value or direction | Number only if Twin-supported; else direction |
| Cause | Traceable cause |
| Evidence | Link to preuve / document / event |
| Confidence | High / medium / low or equivalent |
| Timing | Immediate / delayed / forecast window |
| Affected stakeholder | Who cares |
| Recommended monitoring | What to watch next |

## Precision and forecast rules

- Do not display invented precision.  
- Forecasts MUST be labeled as projections.  
- KPI movement without underlying Twin cause is prohibited.  

## Relationship to modes

| Mode | Panel behavior |
|------|----------------|
| Exploration | Optional preview / “what you would watch” — no official mutation claims |
| Guided Practice | Explanatory impacts; separate from mastery evidence if practice lane |
| Simulation | Full panel for important decisions |
| Evaluation | Panel may be delayed until submit per assessment design |
