# TEC.ERP Cognitive Load and Guidance Fading Standard

> **Status:** CONTRACT — NOT IMPLEMENTED · Wave 1A Research Addendum · Do not treat as shipped UX.

**Authority:** TEC.ERP Learning Operating System Master Contract · Owner review required before Wave 2.

---

## Purpose

Manage intrinsic/extraneous load so professional authenticity does not become cognitive overload. Guidance fades with mastery.

## Learner levels

### NOVICE

- Strong segmentation  
- Worked example  
- Visible process  
- Limited options  
- Guided explanation  
- Immediate feedback  
- Clear stakeholder roles  

### INTERMEDIATE

- Partial example  
- Reduced guidance  
- More investigation  
- Incomplete evidence  
- Realistic exceptions  
- Structured reflection  

### ADVANCED

- Open problem  
- Ambiguous evidence  
- Competing priorities  
- Stakeholder conflict  
- Limited assistance  
- Delayed consequences  
- Executive defense  

## Interface and content rules

1. No first viewport wall of text.  
2. One primary decision at a time.  
3. Progressive disclosure.  
4. Diagrams before long explanation where appropriate.  
5. Glossary on demand.  
6. Details available but not dominant.  
7. No repeated content across cards.  
8. No simultaneous display of every learning element.  
9. No tiny response area for analytical work.  
10. No hidden next step.

## Guidance fading axes

| Axis | Fading behavior |
|------|-----------------|
| Module sequence | Earlier modules more scaffolded; later modules more open |
| Mission sequence | Mission 1 more framing; Mission 3 more judgment |
| Repeated attempts | After attempt N, reduce hints; keep evidence standards |
| Run progression | Later Runs may raise ambiguity under Professor config |
| Professor-controlled mode | Professor can raise/lower guidance without changing Twin truth |

## Mode interaction

| Mode | Guidance posture |
|------|------------------|
| Exploration | Orienting help; no scoring; no irreversible writes |
| Guided Practice | Strongest instructional scaffolding |
| Simulation | Assistance limited by learner level |
| Evaluation | Restricted assistance; immutable evidence |

## Anti-patterns

- Showing BI + AI + theory + all stakeholders + full glossary at once  
- Novice and advanced receiving identical help  
- “Hint spam” that reveals the answer key  
- Advanced tasks that are only longer text, not richer judgment
