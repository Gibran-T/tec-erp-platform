# TEC.ERP Exploration / Guided Practice / Simulation / Evaluation Contract

> **Status:** CONTRACT — NOT IMPLEMENTED · Wave 1A Research Addendum · Do not treat as shipped UX.

**Authority:** TEC.ERP Learning Operating System Master Contract · Owner review required before Wave 2.

---

## Official experience modes

### EXPLORATION

- Free discovery  
- No scoring  
- No irreversible action  
- Safe browsing  
- Enterprise / process / document discovery  
- Stakeholder orientation  

### GUIDED PRACTICE

- Progressive instructions  
- Examples  
- Contextual help  
- Worked steps  
- Explanatory feedback  
- Bounded AI support  

### SIMULATION

- Authentic mandate  
- Limited assistance  
- Persistent Digital Twin consequences  
- Evidence recording  
- Realistic stakeholder communication  
- Learner responsibility  

### EVALUATION

- Controlled environment  
- Restricted assistance  
- Immutable evidence  
- Sequence, diagnosis, interpretation, and judgment assessed  

## Binding UI rules

1. Every route and activity MUST declare its active mode.  
2. The interface MUST never make mode ambiguous.  
3. Mode badge / label is mandatory on learner surfaces.  
4. Mode transitions are explicit (learner or Professor initiated per contract).  

## Mutation and evidence matrix

| Mode | Official Twin mutation | Evidence for mastery | Assistance |
|------|------------------------|----------------------|------------|
| Exploration | None (or sandbox clone only if explicitly labeled; default none) | Orienting only | Orienting |
| Guided Practice | Bounded / reversible or practice lane | Practice artifacts separated from official evidence | Strong |
| Simulation | Yes, governed | Official evidence + consequences | Limited by level |
| Evaluation | Yes, governed; evidence immutable after submit | Assessed evidence | Restricted |

## Help vs evidence

Guided Practice MUST separate instructional help from official mastery evidence. Simulation/Evaluation MUST NOT treat coach text as proof of competence.
