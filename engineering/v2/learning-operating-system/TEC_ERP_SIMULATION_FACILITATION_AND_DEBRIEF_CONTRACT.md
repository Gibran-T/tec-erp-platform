## Purpose

Every serious simulation is taught, not merely played. Facilitation packages are required for every mandat.

## BEFORE package

| Field | Requirement |
|-------|-------------|
| Teaching objective | What the class must learn |
| Prerequisite | Prior modules/missions/knowledge |
| Learner role | Professional identity for the mandat |
| Scenario briefing | Case opening without answer key |
| Time | Planned duration |
| Rules | Mode, assistance limits, collaboration rules |
| Expected evidence | Artifacts learners should produce |
| Likely errors | Anticipated misconceptions |
| Discussion questions | Facilitation prompts ready |

## DURING package

| Field | Requirement |
|-------|-------------|
| Professor observation | What to watch for |
| Decision checkpoints | Moments requiring justification |
| Optional pause | Freeze points |
| Event release | Planned Enterprise Life Engine / scenario events |
| Anonymized class signals | Aggregate progress without shaming |
| Learner assistance limits | By mode and level |
| Intervention triggers | When Professor must step in |

## AFTER package

| Field | Requirement |
|-------|-------------|
| Outcome | What happened in the Twin |
| Consequence analysis | Multi-dimension effects |
| Comparison of decisions | Groups / alternatives |
| Evidence review | Quality of preuves |
| Trade-offs | Accepted risks |
| Misconceptions | Corrected openly |
| Concept connection | Link to academic competency |
| Business transfer | Other enterprise contexts |
| Reapplication task | Short transfer task required |

## Official debrief structure

1. What happened?  
2. Why did it happen?  
3. What did you decide?  
4. What evidence supported it?  
5. What consequence appeared?  
6. What surprised you?  
7. What principle can be extracted?  
8. What would you do differently?  
9. How does this apply elsewhere?

## Professor control capabilities (contract)

Professor MUST be able to:

- Freeze simulation  
- Reveal data  
- Compare groups  
- Replay a decision  
- Trigger an event  
- Hide answer keys  
- Show consequences  
- Collect final synthesis  

## Hard rules

- Every Simulation-mode mandat includes debrief.  
- Answer keys remain Professor-only unless explicitly released.  
- Debrief without reapplication is incomplete (`TEC_ERP_EXPERIENTIAL_REFLECTION_AND_TRANSFER_CONTRACT.md`).
