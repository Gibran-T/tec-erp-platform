# TEC.ERP — Terminologie française institutionnelle

Référence concise pour labels UI / pédagogie. Les codes techniques (`M1-M01`, `SILVER`, `GOLD_M7_M10`, `missionKey`) restent en anglais/ASCII.

| Concept | Préférer | Éviter |
|---------|----------|--------|
| Module de cours | Module M1–M10 | « Chapitre », « Level » |
| Mission métier | Mission | « Exercice » seul |
| Cohorte | Cohorte | « Classe » ambiguë |
| Étudiant | Étudiant / employé junior | « User » |
| Professeur | Professeur | « Teacher » |
| Administrateur | Administrateur | « Admin » dans l’UI FR |
| Certificat Argent | Certificat Argent (Silver) | « Silver » seul en FR |
| Certificat Or | Certificat Or (Gold) | « Gold » seul en FR |
| Capstone | Capstone Équinoxe | « Projet final » vague |
| Évaluation | Évaluation | « Quiz » |
| Déblocage | Débloquer / déblocage | « Unlock » |
| Accès | Accès actif / Accès réservé | « Access » |
| Vérification | Vérification publique | « Verification » |
| Émis / Révoqué | Émis / RÉVOQUÉ | Emis / REVOQUE |
| Tableau de bord | Tableau de bord | « Dashboard » |
| Données de référence | Données de référence | « Master data » en UI |
| Séparation des tâches | Séparation / ségrégation des tâches (SoD) | « SoD » seul sans explication |
| Indicateur | Indicateur / KPI | Mélange non glosé |
| Résumé exécutif | Résumé exécutif | « Executive summary » |

## Accents obligatoires (exemples)

Accès · Évaluation · Émis · Révoqué · Débloquer · Données · Écart · Trésorerie · Définition · Caractères · Pédagogique · Scénario · Intégration · Exécuter

## Règle

Corriger le français visible sans changer les identifiants techniques ni les clés de notation.
